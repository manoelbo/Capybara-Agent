# Feature plans

Plans generated by the **plan-feature** command are saved here.

- **Master plan (passo a passo completo):** `.agents/plan-feature-master.md` — o comando plan-feature executa este plano do início ao fim (Feature Understanding → Codebase + Inspiração mcp_task → Context7 → Estratégia → Geração do plano).
- **Path dos planos:** `.agents/plans/{kebab-case-descriptive-name}.md`
- **Uso:** Comando "Plan Feature" com nome/descrição da feature; o agente segue o plano mestre sem pular etapas.

## Fluxo pós-commit (testes e regressão)

Após implementar (execute-plan) e fazer o commit, rode o comando **post-commit-tests**: ele analisa o último commit, gera ou complementa testes para a feature alterada e atualiza o registro em `.agents/test-registry.md`. O comando **run-tests** usa esse registro para rodar a suíte completa ou apenas os testes da última feature, ajudando a garantir que novas implementações não quebrem o que já estava coberto.
